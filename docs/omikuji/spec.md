# おでかけおみくじ (/omikuji) 仕様書

## 概要

八角形の抽選箱から紙を引くおみくじ風の UI で、行き先（駅 or 観光地）を引き当てる
ミニアプリ。引いた紙は切符・搭乗券に変化する。おでかけガチャの別モードとして提供する。

## 目的

- 「行き先を決める」行為をエンタメ化する
- 大吉/凶のようなランク要素を排除し、出た行き先に格差が生まれないようにする
- API 取得の待ち時間を演出の一部として自然に隠す

## UI 構成

```
┌────────────────────────────────────┐
│ おでかけおみくじ [ガチャ][ダーツ]  │
│                                    │
│  御神籤箱から紙を引いて…          │
│                                    │
│  ┌────────────────────────────┐    │
│  │  ┌──┐                      │    │
│  │  │  │ 八角筒の御神籤箱     │    │
│  │  │  │  （上面に穴）         │    │
│  │  └──┘                      │    │
│  │   [ 引く ]                 │    │
│  └────────────────────────────┘    │
│                                    │
│  [ モード: 駅 / 観光地 ]           │
└────────────────────────────────────┘
```

結果表示時は抽選箱と同じ位置にチケット（切符/搭乗券）へと差し替わる。

## モード

| モード | 結果 | チケット |
|--------|------|----------|
| 駅     | 全国ランダムの駅 | `StationTicket`（国鉄風 硬券） |
| 観光地 | 全国ランダムの観光地 | `BoardingPass`（航空券風） |

どちらも必ず結果を返すため、`fallbackRandomStation` / `fallbackRandomSpot` を使う。

## 状態機械

```
idle ─▶ drawing ─▶ waiting ─▶ extracting ─▶ revealing ─▶ done
 ▲      (0.5s)    (≥1.0s)     (1.5s)         (1.2s)      │
 └──────────────── もう一回 ──────────────────────────────┘
```

| 状態 | 表示 | 期間 |
|------|------|------|
| `idle` | 箱静止 | - |
| `drawing` | 箱が押下フィードバックで軽く凹む | 固定 500ms |
| `waiting` | 箱が揺れ続ける。紙は箱の中に隠れたまま | 1000ms + API 完了待ち |
| `extracting` | 紙が穴から滑らかに出てくる | 固定 1500ms |
| `revealing` | 紙がチケットに展開 | 固定 1200ms |
| `done` | 結果表示＋「もう一回」 | - |

演出の合計最低時間は約 4.2 秒で、通常の API 応答時間を吸収できるよう設計している。
API が遅延した場合は `waiting` が延長されるのみで、紙が半分出かけの状態で待つことがない。

## 切符デザイン

### StationTicket（駅モード）

- 国鉄風の縦長硬券
- 行き先の駅名をセリフ体で強調
- 破線の切り取り線・下部に磁気ストライプ風の帯
- 路線名・都道府県・発行日・4桁の番号

### BoardingPass（観光地モード）

- 航空券風の本券＋ストーブ（半券）分割レイアウト
- `BOARDING PASS` ヘッダ
- `DESTINATION` 欄に観光地名、`REGION` 欄に都道府県
- 観光地カテゴリ（attraction/museum/viewpoint 等）・発行日・4桁番号

## API 信頼性

観光地の都道府県情報は Overpass API の `area["name"="${prefecture}"]->.a` による
OSM 行政境界検索に基づく。県境に跨る観光地で誤表示が起こる可能性は低いと判断し、
都道府県名を切符に載せている。

## エラー処理

- API 失敗時は `sequence` を `idle` に戻し、箱の下に
  「サーバーに接続できませんでした」を表示してリトライ可能にする
- モード切替は実行中無効化（`runningRef`）

## 将来の拡張余地

- ランク要素を持たない別種チケット（乗車券・観光券・参拝券など）の追加
- 着せ替え（抽選箱・紙のバリエーション）
- 履歴（過去に引いたチケットの保管）
